Perspective-based Review

  • Previous: Role-based Review
  • Back to Techniques

Definition

Perspective-based reviews extend role-based reviews by considering different perspectives such as security, usability, and performance. This approach ensures that the software is evaluated from multiple angles, increasing the likelihood of identifying potential issues.

Practical Example

Consider the following code snippet that handles user registration. Review it from a specific perspective, such as security or usability.


// Function to register a new user
function registerUser(username, password) {
    if (username.length < 5) {
        return 'Username must be at least 5 characters long';
    }
    if (password.length < 8) {
        return 'Password must be at least 8 characters long';
    }
    // Simulate user registration
    return 'User registered successfully';
}

// Example usage
console.log(registerUser('user1', 'password123')); // Outputs: 'User registered successfully'
