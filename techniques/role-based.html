Definition

A Role-based review involves evaluating the software from the perspective of different stakeholders, such as a user, developer, or tester. This technique helps ensure that the software meets the needs of all its stakeholders.

Practical Example

Imagine you are part of a team reviewing the login functionality of a website. Review the code snippet below from the perspective of your assigned role (e.g., user, developer, tester).


// Function to validate user login
function validateLogin(username, password) {
    if (!username || !password) {
        return 'Username and password are required';
    }
    // Simulate a login attempt
    return 'Login successful';
}

// Example usage
console.log(validateLogin('user1', 'password123')); // Outputs: 'Login successful'
